# 版本变更日志

## v1.1.0 — 工具集成引擎配置化升级 (2026-03-25)

### 新增功能

#### 主题与引擎升级
- **首页单工具渲染**: `tools` 主题下支持通过 `NEXT_PUBLIC_DEFAULT_TOOL` 环境变量直接将特定工具渲染在首页，并自动屏蔽默认主题的 `TopBanner` 等元素。首页 SEO 自动由目标工具的 `manifest.seo` 提供。
- **自定义导航头部 (`ToolHeader`)**: 工具专属的主题导航，替代了原有的通用导航。支持在 `manifest.json` 通过 `header.logo` 和 `header.nav` 配置专属工具品牌标志及功能菜单。
- **动态版权页脚 (`ToolFooter`)**: 取代默认主题全局页脚的工具级独立脚部组件。支持通过 `manifest.footer.groups` 实现网格形态渲染；若未设，则默认呈现宽幅平铺样式的工具 SEO `title` 与 `description` 信息。
- **配置驱动说明区 (`ToolIntro`)**: 新增 `usage` 字段支持，工具可通过配置 `title`、`description` 和基于 Markdown 的 `content` 来按需显示其详细的使用说明。不支持时不渲染。
- **工具推荐位 (`OtherTools`)**: 新增 `recommendations` 支持配置相关工具流。支持设置模块大标题、副标题，以及可配置打开方式 (`target`) 的推荐工具卡片列表。

#### 数据与配置
- **Manifest 类型扩展**: `type.ts` 内扩展定义了 `ToolHeader`, `ToolFooter`, `ToolUsage`, `ToolRecommendations` 的规范格式。

## v1.0.0 — 工具集成引擎初版 (2026-03-25)

### 新增功能

#### 引擎核心 (`src/core/tooling-engine/`)
- **DynamicLoader**: 多态动态渲染引擎，根据 `manifest.json` 的 `type` 字段自动适配 `component` / `page` 两种渲染模式
- **MetadataRouter**: 自动从工具 manifest 读取 SEO/GEO 信息并注入 Next.js Metadata
- **类型系统**: `ToolManifest`, `ToolSEO`, `ToolGEO` 完整类型定义

#### 主题扩展 (`src/themes/default/`)
- **Tool Layout**: 工具页面专属布局，复用 Header/Footer 并预留工具渲染插槽
- **ToolStage Block**: 工具功能区容器组件
- **ToolIntro Block**: 工具功能介绍图文卡片区
- **OtherTools Block**: 3 列其他工具推荐网格

#### Chat Simulator 工具 (`src/tools/chat-simulator/`)
- **通用聊天数据协议**: 兼容 Discord / WhatsApp / Telegram 等平台的消息模型 (`ChatMessage`, `ChatUser`, `Attachment`, `Reaction`, `MessageGroup`)
- **Zustand 独立 Store**: 零耦合状态管理，内置消息分组 (message stacking) 逻辑
- **脚本解析器**: 支持 `Name: message` 文本格式解析（兼容 DiscordChatSimulator 格式）
- **Headless UI 组件库**: `ChatContainer`, `MessageBubble`, `SenderInfo`, `InputBar`, `AttachmentArea`
- **多平台皮肤基座**: Discord (像素级深色主题), WhatsApp，以及后续 Telegram / Messenger / Instagram / Facebook / Twitter / TikTok 扩展位
- **英文演示数据**: 多用户对话场景

#### 示例工具 (`src/tools/example-tool/`)
- 最小化演示工具，验证 DynamicLoader 挂载功能

#### 路由集成 (`src/app/[locale]/tools/`)
- `layout.tsx`: 工具路由段布局
- `[toolName]/page.tsx`: 动态工具页面，自动 SEO 注入

### 文档
- `docs/技术实施方案.md` — 第一阶段规划文档
- `docs/使用手册.md` — 用户使用指南
- `docs/开发规范.md` — 开发者编码规范

### 依赖变更
- 新增 `zustand` 用于工具独立状态管理
