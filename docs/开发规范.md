# 工具集成引擎 —— 开发规范

## 1. 目录与命名规范

### 1.1 工具目录
- 工具统一存放在 `src/tools/[tool-name]/`
- 工具名使用 **kebab-case**，如 `chat-simulator`、`image-generator`
- 每个工具必须包含 `manifest.json` 和 `index.tsx`

### 1.2 主题目录
- 主题存放在 `src/themes/[theme-name]/`
- 主题下分为 `layouts/`、`blocks/`、`pages/` 三个子目录
- 新增工具相关的块放在 `blocks/` 下，如 `tool-stage.tsx`、`tool-intro.tsx`

### 1.3 引擎核心
- 引擎逻辑存放在 `src/core/tooling-engine/`
- 类型定义在 `types.ts`，渲染加载在 `DynamicLoader.tsx`，SEO 注入在 `metadata.ts`

---

## 2. 代码规范

### 2.1 TypeScript 严格模式
- 所有代码必须符合 TypeScript 严格模式
- 禁止使用 `any`，优先使用 `unknown` 并做类型收窄
- 导出函数和组件必须有明确的类型签名

### 2.2 组件规范
- React 组件使用函数式组件 + Hooks
- 客户端组件必须标注 `'use client'` 指令
- Props 接口定义在组件文件内并使用 `export interface` 导出

### 2.3 样式规范
- **工具内部**：使用 CSS Variables + Tailwind 基础类
- **禁止给 `body`/`html` 注入重置样式**
- 所有 CSS 自定义类名必须带 `ds-` 前缀
- 工具根容器使用 `tool-root-{tool-name}` 类名做沙箱隔离
- 复杂动效使用 `.module.css` 散列化类名

---

## 3. 状态管理规范

### 3.1 零耦合原则
- **禁止工具之间共享状态**
- 每个工具使用独立的 Zustand Store 或 React Context
- Store 创建在 `store/index.ts` 中

### 3.2 Store 结构
```typescript
export const useMyToolStore = create<MyToolState>((set, get) => ({
  // 数据
  data: [],
  
  // 派生数据（缓存）
  computed: [],
  
  // 操作
  setData: (data) => set({ data, computed: derive(data) }),
  reset: () => set(initialState),
}));
```

## 4. 皮肤开发规范

### 4.1 CSS Variables 命名
所有皮肤变量使用 `--ds-` 前缀：
- `--ds-chat-bg` — 聊天背景色
- `--ds-text-primary` — 主文字色
- `--ds-text-secondary` — 副文字色
- `--ds-font-family` — 字体族
- `--ds-border-radius` — 圆角
- `--ds-accent-color` — 强调色（自动引用主题 `--primary-color`）

### 4.2 作用域规则
皮肤 CSS 必须包裹在 `.skin-theme-{name}` 选择器内：
```css
.skin-theme-discord {
  --ds-chat-bg: #313338;
}
.skin-theme-discord .ds-chat-container {
  background-color: var(--ds-chat-bg);
}
```

## 5. Manifest 标准

### 5.1 type 字段
- `"component"` — 工具作为组件嵌入主题布局。主题提供 Header/Footer/Sidebar 等插槽。
- `"page"` — 工具拥有完整页面控制权。主题仅提供基础上下文。

### 5.2 必填与可选字段

| 字段 | 必填 | 类型 | 说明 |
|------|------|------|------|
| `name` | ✅ | string | 工具名称 |
| `version` | ✅ | string | 语义化版本 |
| `type` | ✅ | `"component" \| "page"` | 渲染模式 |
| `entry` | ✅ | string | 入口文件名 |
| `config.skin_preset` | ✅ | string | 默认皮肤 |
| `config.supported_skins` | ✅ | string[] | 支持的皮肤列表 |
| `seo` | ✅ | object | SEO 元数据 |
| `geo` | ❌ | object | GEO 地理元数据 |

---

## 6. 自包含原则

### 6.1 工具层 (`src/tools/`)
- ❌ 禁止使用 `@/` 别名引用框架代码
- ✅ 只允许使用相对导入（`./`, `../`）
- ✅ 只允许引用 npm 包（`react`, `zustand` 等）

### 6.2 主题层 (`src/themes/`)
- ✅ 可以使用 `@/` 别名引用框架共享组件
- ✅ 通过 `getThemeBlock` 加载其他主题块

### 6.3 引擎层 (`src/core/tooling-engine/`)
- ✅ 使用相对导入引用内部模块
- ✅ 使用 `next` 等框架级 npm 包
- ❌ 禁止直接引用具体工具代码
