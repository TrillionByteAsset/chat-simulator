# 工具集成引擎 —— 使用手册

## 1. 快速开始

### 1.1 访问工具页面

启动开发服务器后，访问以下格式的 URL 即可加载任意工具：

```
http://localhost:3000/{locale}/tools/{tool-name}
```

例如：
- `http://localhost:3000/en/tools/chat-simulator` — Chat Simulator 工具
- `http://localhost:3000/en/tools/example-tool` — 示例工具

### 1.2 创建新工具

1. 在 `src/tools/` 下创建一个新文件夹，如 `src/tools/my-new-tool/`
2. 创建 `manifest.json` 配置工具基本信息（参见 §4）
3. 创建 `index.tsx` 作为默认入口组件
4. 访问 `/en/tools/my-new-tool` 即可看到你的工具

引擎会自动读取 `manifest.json`，注入 SEO/GEO 元数据并渲染工具。

## 2. 主题配置

### 2.1 切换到工具主题

在 `.env.local`（或相应的环境变量文件）中设置：

```env
NEXT_PUBLIC_THEME = "tools"
NEXT_PUBLIC_DEFAULT_TOOL = "chat-simulator"
```

设置后，网站首页（`/`）将隐藏默认主题的着陆页与顶部横幅，自动渲染配置的 **默认工具**，直接展示该工具界面与对应配置的组件。如果不配置 `NEXT_PUBLIC_DEFAULT_TOOL`，系统则默认渲染 `chat-simulator`。

### 2.2 可用主题列表

| 主题名称 | 环境变量值 | 说明 |
|----------|-----------|------|
| 默认主题 | `"default"` | Chat Simulator Pro 原始 SaaS 风格首页 |
| 工具主题 | `"tools"` | 工具集合首页，自动展示所有工具入口 |

### 2.3 主题回退机制

当使用 `tools` 主题时，框架会优先加载 `src/themes/tools/` 下的组件。如果某个组件（如 Header、Footer）在 `tools` 主题目录下不存在，会自动回退到 `default` 主题的对应组件，无需重复编写。

### 2.4 工具主题目录结构

```
src/themes/tools/
├── layouts/
│   ├── landing.tsx        # 工具首页布局（挂载 tool-header/tool-footer）
│   └── tool.tsx           # 工具页布局
├── pages/
│   └── dynamic-page.tsx   # 首页渲染逻辑（直接渲染默认工具及其使用文档）
└── blocks/
    ├── tool-header.tsx    # 自定义工具导航头部
    ├── tool-footer.tsx    # 自定义工具底部版权区
    ├── tool-intro.tsx     # 工具使用说明渲染块
    └── index.tsx          # Barrel export
```

## 3. 工具目录结构

每个工具必须遵循以下标准目录布局：

```
src/tools/[tool-name]/
├── manifest.json          # 必须 — 工具配置文件
├── index.tsx              # 必须 — 默认入口组件
├── store/                 # 可选 — 独立状态管理 (Zustand)
│   └── index.ts
├── components/            # 可选 — 工具内部的组件库
├── skins/                 # 可选 — 皮肤 CSS 文件
│   ├── discord.css
│   ├── whatsapp.css
│   └── telegram.css
├── types/                 # 可选 — TypeScript 类型定义
├── utils/                 # 可选 — 工具函数
└── data/                  # 可选 — 模拟数据或静态数据
```

## 4. Manifest 配置详解

`manifest.json` 是工具的身份证，引擎通过它来加载、注册和描述工具。

```json
{
  "name": "My Tool Name",
  "version": "1.0.0",
  "type": "page",
  "entry": "index.tsx",
  "config": {
    "skin_preset": "discord",
    "supported_skins": ["discord", "whatsapp", "telegram"]
  },
  "header": {
    "logo": "/logo.png",
    "nav": [{ "title": "Home", "url": "/" }]
  },
  "footer": {
    "groups": [
      {
        "title": "Resources",
        "items": [{ "title": "Docs", "url": "/docs" }]
      }
    ]
  },
  "usage": {
    "title": "How to Use This Tool",
    "description": "Quick start guide",
    "content": "Markdown content goes here..."
  },
  "recommendations": {
    "title": "Explore More Tools",
    "description": "Discover these related simulation and generator tools.",
    "tools": [
      {
        "title": "WhatsApp Simulator",
        "description": "Create authentic looking WhatsApp chat screenshots.",
        "icon": "💬",
        "url": "/tools/whatsapp-simulator",
        "target": "_blank"
      }
    ]
  },
  "seo": {
    "title": "My Tool Title for SEO",
    "description": "A clear description of what this tool does.",
    "keywords": ["keyword1", "keyword2"]
  },
  "geo": {
    "latitude": "37.7749",
    "longitude": "-122.4194",
    "region": "US-CA",
    "placename": "San Francisco"
  }
}
```

| 字段 | 必填 | 说明 |
|------|------|------|
| `name` | ✅ | 工具显示名称 (Header 默认会显示该名称作为 LOGO 文本) |
| `version` | ✅ | 语义化版本号 |
| `type` | ✅ | `"component"` 或 `"page"` |
| `entry` | ✅ | 入口文件名 |
| `header` | ❌ | 头部导航配置（不存在则不显示导航；logo支持自定义图片） |
| `footer` | ❌ | 底部区配置（不配置时默认横向展示 SEO 标题与描述信息） |
| `usage` | ❌ | 使用文档配置（不配置时不渲染区块；支持 Markdown 格式的 content） |
| `recommendations` | ❌ | 关联工具推荐配置（不配置时不渲染区块；支持自定义标题、副标题及多个工具卡片链接） |
| `config.skin_preset` | ✅ | 默认皮肤名称 |
| `seo` | ✅ | SEO 元数据（Footer 默认会回退显示此处的 title/description） |
| `geo` | ❌ | GEO 地理位置元数据 |

## 5. 皮肤系统

### 5.1 皮肤切换

皮肤通过 CSS Variables 实现。DynamicLoader 会自动为工具容器注入 `skin-theme-{preset}` 类名。

CSS 中通过作用域选择器控制皮肤：

```css
.skin-theme-discord {
  --ds-chat-bg: #313338;
  --ds-text-primary: #dbdee1;
}

.skin-theme-whatsapp {
  --ds-chat-bg: #e5ddd5;
  --ds-text-primary: #111b21;
}
```

### 5.2 自定义皮肤

1. 在 `skins/` 目录下创建新的 CSS 文件（如 `telegram.css`）
2. 定义 `.skin-theme-telegram` 选择器及对应的 CSS Variables
3. 在 `manifest.json` 的 `supported_skins` 中添加 `"telegram"`
4. 在入口文件中 `import './skins/telegram.css'`

## 6. 工具导出与热插拔

### 6.1 导出工具

将 `src/tools/[tool-name]/` 整个文件夹复制即可。该工具的所有依赖（组件、状态、皮肤、类型）均自包含在此目录内。

### 6.2 导入工具到新项目

1. 确保新项目同样基于 Chat Simulator Pro 框架
2. 将工具文件夹粘贴到 `src/tools/` 目录下
3. 安装工具所需的 npm 依赖（如 `zustand`）
4. 访问 `/tools/{tool-name}` 即可使用

## 7. 页面布局说明（UE 线框）

### 7.1 工具首页布局（`NEXT_PUBLIC_THEME = "tools"` 时的首页）

当启用 tools 主题时，首页不再展示通用入口，而是**直接拉起默认工具**。

| 区域 | 对应组件 | 说明 |
|------|---------|------|
| **Header** | `tools/blocks/tool-header.tsx` | 工具自定义导航（展示设定的 logo 或应用名，若无 `header.nav` 配置则仅显示 LOGO） |
| **ToolStage** | `default/blocks/tool-stage.tsx` | 工具主渲染区，运行默认设定的核心工具体验 |
| **ToolIntro** | `tools/blocks/tool-intro.tsx` | 工具使用指南区块，由 manifest `usage` 配置驱动解析 Markdown，未配置则不渲染 |
| **OtherTools** | `tools/blocks/other-tools.tsx` | 推荐工具网格区块，由 manifest `recommendations` 配置驱动，未配置则不渲染 |
| **Footer** | `tools/blocks/tool-footer.tsx` | 工具页脚，未配置 `footer.groups` 时横向流式展示 SEO Title / Desc，配置后呈右侧网状菜单 |

### 7.2 单个工具详情页布局（`/tools/{tool-name}`）

| 区域 | 对应组件 | 说明 |
|------|---------|------|
| **Header** | `default/blocks/header.tsx` | 品牌 Logo 和导航 |
| **工具功能区** | `default/blocks/tool-stage.tsx` | 工具主渲染区，由 DynamicLoader 注入 |
| **工具功能介绍** | `default/blocks/tool-intro.tsx` | 图文介绍功能特色卡片 |
| **其他工具** | `default/blocks/other-tools.tsx` | 3 列推荐工具卡片网格 |
| **Footer** | `default/blocks/footer.tsx` | 页脚 |
